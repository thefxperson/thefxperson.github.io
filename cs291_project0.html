Why are there diminishing returns at higher concurrency levels?

At higher concurrency levels, the number of requests likely is overwhelming the server, or the server is rate limiting the benchmarking tool.

What’s the performance difference when requesting HTTP and HTTPS?

Requesting HTTPS is approximately half as efficient as requesting via HTTP. For 16 concurrent HTTPS requests, GitHub completed 2775, 44% of the number completed via HTTP.

How can github respond so quickly?

GitHub can quickly respond due to having multiple servers to process requests. Their servers likely cache recent requests, which allows a rapid respose without having to fetch the resources from slow storage again
